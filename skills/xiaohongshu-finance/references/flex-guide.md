# Flex 模板排版指南

## 核心原则

flex 模板的核心理念是 **自由发挥**，不设固定布局。AI 根据内容特点自行决定排版方式。
以下只提供配色参考和基本质量要求，不限制具体实现。

---

## 配色参考（深色主题）

| 用途 | 色值 |
|------|------|
| 标题/高亮文字 | `#EAE0CF` |
| 正文/数据 | `rgba(234,224,207,0.75)` |
| 标签/辅助色 | `#7288AE` |
| 次要文字 | `rgba(234,224,207,0.4)` |

| 区块背景 | `rgba(75,86,148,0.08)` |
| 标签底色 | `rgba(114,136,174,0.15)` |

## 质量要求（必须遵守）

### 1. 信息密度

一张 flex 图必须包含 **至少 8 个有效信息点**，不能只有标题加两三行文字。以下为最低标准：

- 有 KPI 数据卡片（2-4 个核心数字）
- 有数据表格或对比区块（至少 4 行数据）
- 有结构化分析（分点、标签、分类展示）
- 有风险提示或数据来源说明

### 2. 可视化元素

纯文字段落不得超过内容的 30%。每张图必须包含至少 2 种可视化形式：

- KPI 数据卡片（数字 + 涨跌色）
- 数据表格（带正负着色）
- 左右 / 上下对比区块
- 标签云 / Chip 分类
- 进度条 / 占比条
- 图表（通过 driver.py 另生成 SVG 嵌入）

### 3. 排版要求

- 字号层级分明：标题 36-48px，章节标题 20-24px，正文 16-20px，辅助文字 13-15px
- 区块间距清晰：margin 16-24px，padding 14-22px
- 圆角统一：12px（卡片），16px（标签）
- 使用分隔线（`height:1px; background:linear-gradient(...)`）划分章节
- 涨跌数据必须着色，不能只用文字描述

### 4. 内容要求

- 数据必须有真实来源，不能编造数字
- 涉及投资必须包含风险提示
- 不要使用 emoji

## 常见内容类型的布局参考

AI 根据内容类型自由选择排版方式，以下仅举例：

### 行情分析类（如 A股黑色星期五）

参考结构：KPI 卡片（3 个指数）→ 行业涨跌表（6 行）→ 原因/策略/资金三栏对比

### 行业深度类（如商业航天）

参考结构：KPI 卡片（涨停数/涨幅/市值）→ 对比区块（新旧对比）→ 产业链标签云 → 风险提示

### 数据对比类（如基金定投）

参考结构：对比表格 → 数据表格 → 趋势要点 → 结论

### 热点解读类（如政策发布）

参考结构：KPI 卡片（关键数据）→ 左右对比（新旧政策）→ 受益方向标签 → 风险提示

---

## 注意事项

- 不要写死具体布局，AI 根据内容自由选择最合适的表达方式
- 信息密度是硬性要求，不能偷懒只放少量文字
- 可视化是硬性要求，不能只有文字
- 配色使用主色板，不要引入额外颜色
