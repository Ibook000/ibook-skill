# Self Rules

## Global Rules

- 先做最小可运行版本，再优化
- 不要停在想法阶段，尽快闭环
- 复杂问题必须拆模块
- 能自动化就不要手动重复
- 能工具化就不要只靠 prompt
- 配置和代码必须分离
- 默认加日志
- 默认考虑异常处理
- 默认考虑长期运行

## Working Rules

- 不要为了炫技把问题做复杂
- 优先把核心路径跑通
- 小项目先完成闭环，大项目再分层
- 回答问题时要给明确结论
- 方案要尽量能直接复制执行

## Product Rules

- 降低用户上手门槛
- 提供 README
- 提供配置模板
- 尽量给快速启动方式
- 常见故障要能快速定位

## Trading Rules

- 不频繁开单
- 不重仓赌方向
- 不靠情绪平仓
- 方向对了要有拿单计划
- 踏空后不追
- 浮盈不是离场理由，规则才是
- 小赚小亏大赚，绝不大亏
- 重仓必须有更强胜率依据
- 在犹豫要不要跑的时候，优先执行保护动作

## Growth Rules

- 目标要写下来
- 学习不能只靠一时热血
- 输入要持续，包括阅读与语言能力提升
- 记录不是为了好看，是为了推动成长
