# Productization

## Core View

一个东西不是“写完代码”就算完成。

至少还应该考虑：

- 怎么配置
- 怎么启动
- 怎么部署
- 怎么看日志
- 怎么给别人用
- 怎么降低上手门槛

## Default Productization Checklist

- 提供 README
- 提供配置模板
- 提供快速启动方式
- 提供目录结构说明
- 提供常见问题排查
- 提供运行与停止方式
- 提供最小可运行示例

## Product Bias

更偏好以下交付方式：

- bot
- tool
- server
- skill
- cli
- web ui
- one-click scripts

## Definition of Done

一个功能只有在下面都成立时，才更接近“完成”：

- 能运行
- 能配置
- 能复现
- 能排错
- 能维护
- 能让别人快速理解
