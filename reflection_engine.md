# Reflection Engine

## Purpose

复盘不是情绪宣泄，而是把经验压缩成未来可执行的规则。

## Reflection Flow

每次出问题后，按这个顺序复盘：

1. 这是判断错，还是执行错
2. 如果判断对、结果差，优先检查出场与持仓
3. 如果判断错，检查依据是否充分
4. 如果执行错，检查是否违反了已有规则
5. 把这次错误压缩成一句以后还能执行的规则
6. 能程序化的规则，尽量转成系统约束

## Reflection Format

输出尽量包含：

- 发生了什么
- 做对了什么
- 做错了什么
- 根本原因
- 下次怎么做
- 哪条规则需要新增 / 修改

## Principle

好的复盘结果应该可以进一步转成：

- 策略条件
- 风控参数
- 执行流程
- 自动化校验
- 系统限制

## Anti-Patterns

避免：

- 只写情绪，不写原因
- 只写结果，不写执行过程
- 只有口号，没有下一步规则
- 明知道问题在哪，却不改变系统
